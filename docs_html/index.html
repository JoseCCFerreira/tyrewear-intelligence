Predictive tyre analytics

Predictive tyre analytics for safety, durability and cost optimization.

A portfolio-grade platform for tread depth, wear rate, statistics, clustering, ML, maps and recommendation.

Problem

Choosing tyres only by price ignores durability, wet safety, rolling resistance, noise and cost per kilometre.

Solution

The platform combines a realistic dataset, SQLite, DuckDB, dbt, statistical testing, clustering, machine learning, Streamlit and an AI advisor MVP.

Tread Depth Methodology

Wear: TD initial - TD current. Wear rate: wear / mileage × 10,000. Remaining life: max(TD current - 1.6, 0) / wear rate × 10,000.

Statistical Methods

Normality, p-value interpretation, parametric tests, non-parametric tests, correlations and regression are used to separate real signals from noise.

Clustering

K-Means, DBSCAN, GMM and PCA reveal natural tyre groups by TD, wear, cost and remaining life. UMAP/t-SNE are roadmap items.

Machine Learning

Models predict remaining life, wear rate and risk class. Classical tabular models are the baseline before deep learning.

TensorFlow and PyTorch

They add real value for images, tabular neural-network comparison, time series and recommendation embeddings, not for every simple dashboard task.

Geographic Intelligence

The app maps TD, wear rate and risk share by European country, region and city with interactive filters.

How to Run

pip install -r requirements.txt
python3 scripts/run_full_pipeline.py
streamlit run app.py --server.port 8580
python3 -m http.server 8570

# dbt: use a separate environment
python3 -m venv .venv-dbt
source .venv-dbt/bin/activate
pip install -r requirements-dbt.txt
cd dbt_tyre_warehouse
dbt build --profiles-dir .
