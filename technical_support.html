1. Overview

What the project is trying to prove.

TyreWear Intelligence treats tyre selection as a data problem. Instead of ranking tyres only by price, it compares technical and economic signals: tread depth, durability, wear rate, safety, noise, energy efficiency and cost per kilometre.

The core question is simple: which tyre gives the best long-term value without sacrificing safety or performance?

The pipeline separates responsibilities so that each stage can be tested and explained. Public sources feed a transactional database, analytical tables are built locally, statistical and ML routines generate insight, and the dashboard presents the result to a user.

6. Statistics

How to compare tread depth and tyre groups correctly.

Statistics is used to decide whether observed differences between tyres are likely to be meaningful. For example, if Brand A has a lower average wear rate than Brand B, the project should test whether that difference is statistically supported.

TestQuestion answeredWhen to use
Normality testDoes the variable look normally distributed?Before choosing parametric tests.
t-testAre two group means different?Two groups, approximately normal data.
ANOVAAre three or more group means different?Multiple groups, approximately normal data.
Mann-WhitneyAre two distributions different?Two groups, non-normal data.
Kruskal-WallisAre multiple distributions different?Multiple groups, non-normal data.
p-valueHow surprising is the result if there is no real difference?Decision support, not absolute proof.
Practical rule: if p-value is below the chosen alpha, commonly 0.05, the result is treated as statistically significant. Still, significance should be interpreted with effect size, sample size and business context.

7. Machine Learning

Prediction is useful only when validation is honest.

ML models are trained to predict tyre life, wear rate and risk. The target can be continuous, such as expected kilometres, or categorical, such as high/medium/low wear risk. The model should always be evaluated on unseen data.

TaskTargetUseful metrics
RegressionExpected tyre life or wear rate.MAE, RMSE, R2.
ClassificationRisk class or recommendation class.Accuracy, F1, ROC AUC.
Feature importanceWhich factors influence prediction.Permutation importance, tree importance, SHAP.
Deep learningComplex patterns in larger datasets.Validation loss, MAE, F1.
 
# Example model workflow
X = tyre_features
y = expected_life_km

X_train, X_test, y_train, y_test = train_test_split(X, y)
model.fit(X_train, y_train)
predictions = model.predict(X_test)

mae = mean_absolute_error(y_test, predictions)

8. Charts

What each graph should explain.

Charts are not decoration. Each chart should answer a question and make the result easier to defend. The dashboard should combine exploratory views, comparison views and model explanation views.

9. AI Advisor

Turning analysis into answers.

The AI advisor is a layer that answers questions about tyres using the project outputs. It should not invent data. It should answer from available tables, model results, statistical tests and documented business rules.

Example question: “Which tyre gives the best value for 225/45 R17 if I prioritize wet safety and low cost per kilometre?”

10. Results

How to interpret the output.

The final result should not be a single “best tyre” without context. A strong output explains the trade-off between safety, durability, cost and uncertainty.

Best value Lowest cost per kilometre after durability adjustment.
Safest option Best wet grip and lower wear risk under constraints.
Longest life Highest predicted useful kilometres before replacement threshold.
Recommended tyre Balanced score across cost, safety, durability and noise.

How to run and validate the project.

source .venv/bin/activate
pip install -r requirements.txt

# 2. Load transactional database

python scripts/build_duckdb.py

python scripts/statistics_tests.py
python scripts/train_models.py

# 5. Start dashboard
streamlit run app.py
